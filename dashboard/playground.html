Under the Hood: Graph RCA Architecture

How Cognee constructs and queries codebase time-travel graphs to pinpoint post-release bugs.

Phase 1: Code Baseline Extraction

📁 ──▶ 🌲 ──▶ 🕸️

The code is parsed using Tree-sitter into abstract syntax trees (ASTs). Rather than storing the raw syntax, RCA extracts high-level logical nodes using Cognee-native models:

  • CodeFile: Represents source files and scopes.
  • FunctionDefinition: Represents individual functions, recording name and line ranges.
  • ClassDefinition: Represents classes.
  • ImportStatement: Tracks dependencies between files.

Call Edge Resolution: Identifies calls from one function to another within the same file or resolved via static import references. Nodes are assigned stable UUIDs derived from their qualified path (e.g. uuid5("type:file:symbol")), making ingestions idempotent.

Phase 2: Git Diff Syncing

🐙 ──▶ 📏 ──▶ 🏷️

When a release commits, we run git diff prev..rel. The line ranges of changes are mapped back to enclosing AST elements at both versions:

  • Modified/Added Symbols: Linked directly to the function node with a MODIFIED_IN edge pointing to a Release node.
  • Deleted Symbols: Deletions break callers. Because the symbol node no longer exists in the new code baseline, a ChangedSymbol is mapped to the containing CodeFile.
  • Miss Fallbacks: If a git diff line cannot be mapped to a specific function, it falls back to the parent file so that changes are never silently ignored.

Phase 3: Traversal & Scoring

🔍 ──▶ 🌐 ──▶ 📊

When a stack trace is submitted, the analysis proceeds entirely locally and deterministically without LLM lag:

  1. Trace Parsing: Extracts files and symbols from the frames (starting from the deepest frame).
  2. Seed Nodes Lookup: Matches frames against baseline nodes in the Cognee database.
  3. k-Hop Traversal: Performs a multi-directional BFS traversal over calls, imports, and containment edges.
  4. Ranking: Candidates in the release window are scored based on proximity:
    • 0-hop dominates: Changed functions directly in the stack trace score highest.
    • Distance: Scores decrease exponentially with hop distance.
    • Recency: Newer releases out-rank older releases.
    • Change Type: Modified/Deleted symbols out-rank Added symbols.

Quickstart Documentation

Set up Cognee Release Root-Cause Analysis (RCA) on your own codebase in minutes.

1. Installation

Install the required packages. We recommend using uv or pip in a virtual environment.

# Install core Cognee along with tree-sitter grammars and supporting packages
pip install "cognee[codegraph]==1.2.2" tree-sitter-javascript tree-sitter-typescript networkx pytest

2. Initialization

Configure Cognee to run locally. This configures a local Sqlite database and filesystem store.

import asyncio
from rca import env
from rca.baseline import build_baseline
from rca.diff_sync import sync_release
from rca.query import analyze

async def main():
    # 1. Initialize local graph storage
    await env.init()

    # 2. Build the baseline code graph (reads files and constructs AST relationships)
    await build_baseline("/path/to/my/project")

    # 3. Synchronize a release diff (tags git changes onto the code graph)
    await sync_release(
        repo_path="/path/to/my/project",
        prev_sha="a1b2c3d4",
        rel_sha="e5f6a7b8",
        tag="v1.1.0",
        timestamp="2026-07-01T12:00:00Z"
    )

    # 4. Analyze an incident traceback
    findings = await analyze(
        stack_trace="""Traceback (most recent call last):
  File "app/api.py", line 9, in handle_checkout
    return process_payment(order)
ValueError: invalid amount""",
        k_hops=2,
        release_window=["v1.1.0"] # filter by release window
    )

    for f in findings:
        print(f"culprit: {f.qualified_name} (score: {f.score})")
        print(f"diff hunk:\n{f.diff_hunk}")

if __name__ == "__main__":
    asyncio.run(main())
